# دليل الهوية البصرية الخاصة بالأطباء

## نظرة عامة
تم تطوير نظام يسمح لكل طبيب بتخصيص هويته البصرية الخاصة داخل التطبيق. يمكن لكل طبيب الآن التحكم في:

### 1. الألوان
- **اللون الأساسي**: يظهر في الأزرار والعناوين والروابط
- **اللون الثانوي**: لون مكمل للهوية البصرية
- **لون التمييز**: للعناصر البارزة
- **لون الخلفية**: خلفية التطبيق
- **لون النص**: لون النصوص الرئيسية

### 2. الخطوط
- **خط العناوين**:用于العناوين الرئيسية
- **خط النصوص**: للنصوص العادية

### 3. الأنماط
- **شكل الأزرار**: دائرية، مربعة، أو حبوية
- **شكل البطاقات**: مع ظل، مع إطار، أو بسيطة

### 4. الشعار
- رفع شعار خاص بالعيادة
- يظهر في الشريط الجانبي وصفحة تسجيل الدخول

### 5. إعدادات الروشتة
- عنوان العيادة
- رقم هاتف العيادة
- تعليمات الروشتة الافتراضية

## كيفية الاستخدام

### 1. تنفيذ قاعدة البيانات
```sql
-- قم بتشغيل ملف DOCTOR_BRANDING_SIMPLE.sql
-- لإضافة أعمدة الهوية البصرية إلى جدول الأطباء
```

### 2. إنشاء حقول التخزين
في Supabase Dashboard → Storage، قم بإنشاء:
- `doctor-logos` - لشعارات الأطباء
- `doctor-watermarks` - لعلامات الماء المائية (اختياري)

### 3. استخدام النظام
1. سجل الدخول كطبيب
2. اذهب إلى صفحة الإعدادات
3. اختر "المظهر والهوية"
4. قم بتخصيص الألوان والخطوط والأنماط
5. ارفع شعار العيادة
6. احفظ الإعدادات

## المكونات المحدثة

### BrandingContext.tsx
- تم تحديثه لجلب بيانات الهوية البصرية الخاصة بالطبيب الحالي
- يدعم تحديث جميع خيارات التخصيص
- يتكامل مع PowerSync للمزامنة غير المتصلة بالإنترنت

### Settings.tsx
- واجهة شاملة لتخصيص الهوية البصرية
- أدوات اختيار الألوان
- قوائم منسدلة للخطوط والأنماط
- معاينة مباشرة للتغييرات

## قواعد الأمان
- كل طبيب يمكنه فقط تعديل بياناته الخاصة
- الصور تُرفع في مجلدات خاصة بمعرّف المستخدم
- تطبيق سياسات RLS لحماية البيانات

## المزامنة
- التغييرات تُحفظ محلياً فوراً
- تُزامن مع Supabase عند الاتصال بالإنترنت
- تدعم العمل Offline أولاً
